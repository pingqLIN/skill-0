## 執行摘要

### 什麼是 Skill-0？

Skill-0 是一個**專門的技能分解與分析框架**，使用三元分類系統（Actions、Rules、Directives）來解析和結構化 AI 技能，特別是 Claude Skills 和 MCP Tools。

### 關鍵差異化特徵

| 方面 | Skill-0 | 相似專案 |
|------|---------|----------|
| **主要焦點** | 技能分析與分解 | 執行、發現或收集 |
| **方法論** | 語義三元分類 | 模式匹配或基於規則 |
| **輸出** | 結構化 JSON (Schema v2.2.0) | 各異（Markdown、配置、無） |
| **搜尋** | 基於向量的語義搜尋 | 基於文字或無搜尋 |
| **目標使用者** | 技能架構師、分析師 | 開發者、終端使用者 |

## 競爭優勢

### Skill-0 最擅長的

#### 1. **三元分類**

```json
{
  "decomposition": {
    "actions": [
      {
        "id": "a_001",
        "name": "讀取 PDF",
        "action_type": "io_read",
        "deterministic": true,
        "side_effects": ["記憶體配置"]
      }
    ],
    "rules": [
      {
        "id": "r_001",
        "name": "檢查檔案是否存在",
        "condition_type": "existence_check",
        "output": "boolean"
      }
    ],
    "directives": [
      {
        "id": "d_001",
        "directive_type": "completion",
        "description": "所有表格已提取",
        "decomposable": true
      }
    ]
  }
}
```

**為何重要：**
- **Actions** 是原子操作（構建塊）
- **Rules** 是決策點（流程控制）
- **Directives** 是目標（成功標準）

這實現了：
- 自動化複雜度分析
- 依賴圖形化
- 可重用性評分
- 安全性分析

---

#### 2. **語義搜尋與上下文**

**傳統搜尋（基於關鍵字）：**
```
查詢：「PDF 處理」
結果：僅精確匹配
```

**Skill-0 搜尋（語義）：**
```
查詢：「PDF 處理」
結果：
  1. "Docx Skill" (89%) - 相似的文件處理
  2. "Excel Skill" (76%) - 表格數據提取
  3. "圖像 OCR" (71%) - 視覺文件解析
```

**為何重要：**
- 發現您不知道存在的相關技能
- 理解領域概念
- 跨語言/術語工作

---

#### 3. **自動模式提取**

Skill-0 自動識別：

```python
{
  "patterns": {
    "action_combinations": [
      {
        "pattern": ["io_read", "transform", "io_write"],
        "frequency": 12,
        "skills": ["pdf", "docx", "xlsx"]
      }
    ],
    "directive_usage": [
      {
        "type": "completion",
        "frequency": 30,
        "common_phrases": ["已提取", "已處理", "已儲存"]
      }
    ]
  }
}
```

**為何重要：**
- 自動識別最佳實踐
- 建議技能改進
- 啟用技能模板生成

### 最終結論

**Skill-0 補充生態系統**，通過提供缺失的分析層。它不與現有工具競爭，而是通過以下方式使它們更好：

- 更好的技能發現（語義搜尋）
- 更好的技能質量（自動化分析）
- 更好的技能開發（結構化模板）
- 更好的技能執行（驗證分解）
